Configuración y respaldo

Cómo usar este archivo en iPhone

En iPhone, Safari no aparece como opción para "Abrir con" un archivo .html guardado en Archivos (es una limitación de iOS). Por eso, la forma más práctica de usar esta app todos los días es con la app gratuita Documents by Readdle, que tiene su propio navegador y sí permite abrir y guardar datos en archivos locales.

  1. Instalá Documents by Readdle desde el App Store (gratis).
  2. Guardá ondina-travel.html dentro de Documents (podés moverlo desde la app Archivos o guardarlo ahí directamente).
  3. Tocá el archivo: se abre el navegador interno de Documents con la app funcionando completa (gráficos, formularios y guardado local).
  4. Tocá el ícono de marcador/estrella para guardarlo como acceso rápido dentro de Documents.

Alternativa para Mac (o si preferís usar Safari/Chrome de verdad): subí este archivo a un hosting gratuito (por ejemplo GitHub Pages) para tener un link https:// propio. Una vez con ese link, abrilo desde Safari en iPhone o Mac, y usá "Añadir a pantalla de inicio" para tener un acceso directo — ahí sí todo funciona como una app normal, sin las restricciones de los archivos locales.

Si ves la advertencia roja arriba de la pantalla, significa que en este modo no se puede guardar de forma permanente. Exportá un respaldo en JSON antes de cerrar la pestaña.

Sincronización en la nube (todos los dispositivos)

Estado actual: Modo local (sin sincronizar)

Para que la Mac, el iPhone y cualquier otro dispositivo vean los mismos files, clientes y movimientos en tiempo real, conectá esta app a una base de datos gratuita (Firebase). Necesitás un "código de conexión" (un texto en formato JSON) — lo generás una sola vez y lo pegás en cada dispositivo que quieras sincronizar.

¿Todavía no tenés un código de conexión? Generarlo (una sola vez, desde cualquier dispositivo)
  1. Entrá a console.firebase.google.com con una cuenta de Google y creá un proyecto gratis (cualquier nombre, por ejemplo "ondina-travel").
  2. En el proyecto, tocá el ícono "</>" (Web) para agregar una app web. Ponele un nombre y registrala. Vas a ver un bloque de código con valores como apiKey, authDomain, etc. — copialos en los campos de abajo.
  3. En el menú lateral, andá a Build → Firestore Database → "Create database" → elegí cualquier ubicación → modo "production".
  4. En la pestaña "Rules" de Firestore, reemplazá el contenido por: 
    rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /ondina_travel_sync/{doc} {
      allow read, write: if request.auth != null;
    }
  }
}
    y tocá "Publish".
  5. En el menú lateral, andá a Build → Authentication → "Get started" → en la pestaña "Sign-in method", habilitá "Anonymous".
  6. Completá los campos de abajo con los datos que copiaste, elegí un nombre para tu agencia y, si querés, un PIN de acceso. Tocá "Generar código de conexión" y después "Conectar este dispositivo".
  7. Para los demás dispositivos: abrí la app, pegá el mismo código de conexión generado, y tocá "Conectar este dispositivo".

Backup automático a Google Drive

Guardá una copia del backup directamente en tu Google Drive. Se crea un archivo ondina-travel-backup-FECHA.json en la carpeta que elijas.

Para activarlo necesitás pegar acá tu Client ID de Google (se configura una sola vez). Clic en "¿Cómo obtenerlo?" para ver las instrucciones.

¿Cómo obtener el Client ID? (5 minutos, una sola vez)
  1. Entrá a console.cloud.google.com con tu cuenta de Google.
  2. Creá un proyecto nuevo (cualquier nombre, ej: "ondina-backup").
  3. En el menú lateral → APIs y servicios → Biblioteca → buscá "Google Drive API" → habilitala.
  4. Andá a APIs y servicios → Credenciales → "+ Crear credencial" → ID de cliente de OAuth.
  5. Tipo de aplicación: Aplicación web. En "Orígenes autorizados" agregá la URL de tu GitHub Pages (ej: https://tu-usuario.github.io).
  6. Copiá el ID de cliente que aparece (termina en .apps.googleusercontent.com) y pegalo arriba.
  7. En Pantalla de consentimiento OAuth → agregá tu email como "usuario de prueba".
